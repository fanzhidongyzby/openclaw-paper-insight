# 论文公众号解读技能

将学术论文转化为**微信专业公众号风格**的深度解读文章。

## 功能简介

融合学术解读和公众号传播两个流派，生成高质量的论文解读：

- **原子管线设计**：split → squeeze → plain → feynman → napkin sketch → advisor review
- **零术语规则**：承重概念必须场景化（3 级渐进），认知碰撞卡片用 ASCII art 直观展示洞见
- **公众号风格**：手机友好阅读，对话式表达，emoji、加粗、分隔线等视觉元素，故事性，数据支撑

## 使用方式

当分享 arxiv 链接、论文 URL、PDF，或要求解读论文时触发。

**支持的输入格式**：

| 输入 | 示例 |
|------|------|
| arxiv ID | `2601.01290` |
| arxiv URL | `https://arxiv.org/abs/2601.01290` |
| HTML URL | `https://arxiv.org/html/2601.01290` |
| PDF 路径 | `~/papers/example.pdf` |


## 核心原则

### 零术语规则

承重概念必须场景化（3 级渐进）：

1. 先找读者亲身经历过的场景
2. 在场景中逐步展开（至少 3 级）
3. 最后才贴技术名字

**判断标准**：去掉技术名字，读者仍然知道你在说什么 = 落地成功。

### 认知碰撞卡片

ASCII art 直观展示论文洞见与个人认知的碰撞。

**碰撞形状**：分叉型、张力型、阈值型、缺口型、翻转型——选跟碰撞匹配的空间结构。

**好卡片标准**：遮住文字只看线条，仍能感受到结构关系。

### 公众号风格

- 📱 标题层级清晰，适合手机阅读
- 💬 流畅自然，对话式表达
- 🎨 视觉丰富，emoji、加粗、分隔线
- 📖 故事性，有吸引力的前言
- 📊 数据支撑，完整表格、实验数据
- 🎯 深入浅出，生活化类比
- 💡 互动感，用问句让读者有参与感

## 设计原则

- 深度解读，不是简单摘要
- 实验数据必须完整
- 每个技术概念都有生活场景
- 有启发性碰撞卡片（如有）
- 公众号风格，emoji、对话式、视觉丰富
- 永远使用中文表达
- 零割裂感：输出是一篇连贯的认知旅程
